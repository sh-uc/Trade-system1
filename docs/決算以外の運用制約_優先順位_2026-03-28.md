# 決算以外の運用制約 優先順位（2026-03-28）

## 1. 目的

暫定運用候補セット `swing_v1_candidate9` を前提に、  
決算回避以外で、次にどの運用制約を入れるべきかを整理する。

ここでいう「運用制約」は、  
売買ロジックを変えるのではなく、**実運用で必要な安全弁や制限** を指す。

## 2. 現時点の前提

- 9銘柄版は `paper simulation` で有意に強い
- `FY only block` は導入しても全体には大きく崩れていない
- 一方で、まだ本番寄りの制約は十分に入っていない

したがって次の段階では、  
「さらに銘柄を増やす」より、  
**本番寄りにした時にまだ耐えるか** を先に見る方が自然である。

## 3. 推奨優先順位

### 優先1. 同時保有数 / 1日新規建て数の上限

内容:
- `MAX_OPEN_POSITIONS`
- `MAX_NEW_POSITIONS_PER_DAY`

理由:
- 実運用で最も効く制約のひとつ
- シグナルが重なった日に過剰に建ちすぎるのを防げる
- ポートフォリオ観点の挙動が見やすい

現時点のおすすめ:
- 同時保有上限 `4 - 5`
- 1日新規上限 `2`

初回の確認値:
- `MAX_OPEN_POSITIONS=5`
- `MAX_NEW_POSITIONS_PER_DAY=2`

### 優先2. イベント以外の手動停止フラグ

内容:
- 銘柄別停止
- 全体停止
- 強制クローズ

理由:
- 「急な予測不能事象」に備えるために必要
- 実運用では、ロジックよりこの逃げ道の有無が重要になる

実装イメージ:
- `ticker_events` に `manual_stop`
- または `risk_control_flags` 的な別テーブル

### 優先3. 権利付最終日 / 権利落ち回避

内容:
- 配当や優待の権利取り前後で新規停止
- 必要なら強制決済

理由:
- 株価ギャップや需給の歪みが出るため
- 決算回避と同じ発想で扱いやすい

補足:
- `yfinance` の `ex_dividend` は補助には使える
- ただし日本株の正式運用には別ソース検討余地あり
- 初回評価では、`EX_DIVIDEND_BLOCK_DAYS=1/2/3` が同一だった
- 一方で `EX_DIVIDEND_FORCE_CLOSE_DAYS=1` はやや強すぎた
- 詳細: `docs/権利落ち回避_初回評価_2026-03-28.md`

### 優先4. ポジションサイズの総量制御

内容:
- 総投下資金上限
- 1セクターあたり上限
- 1銘柄あたり上限

理由:
- 今は `PER_TRADE` ベースなので、同系統銘柄に偏る可能性がある
- 特に `機械`, `医薬`, `情報通信` の偏りは今後見たい

初回評価:

- `MAX_TOTAL_EXPOSURE=1,500,000` は軽く効いた
- `MAX_TOTAL_EXPOSURE=1,000,000` は削りすぎた
- ただし現時点では、まだ標準制約として必須とは言いにくい
- 暫定運用候補セットでは、今回は見送りとする
- 詳細: `docs/総量制御_初回評価_2026-03-28.md`

### 優先5. 異常値日の新規停止

内容:
- ギャップが大きすぎる日は見送る
- 当日ボラが閾値超なら停止

理由:
- 既存ロジックにも `GAP_ENTRY_MAX` はあるが、
  ポートフォリオ運用では別レイヤでの安全弁にも意味がある

## 4. 現時点で後回しでよいもの

### セクター別完全配分最適化

まだ早い。  
まずは同時保有上限と銘柄別制御の方が価値が高い。

### 複雑な地合いフィルタ

面白いが、制約というよりロジック拡張寄り。  
今は運用安全弁の方を優先したい。

## 5. 直近の実装順

おすすめ順は次です。

1. 同時保有数 / 1日新規建て数の上限
2. 手動停止フラグ
3. 権利付最終日 / 権利落ち回避
4. 総量制御

## 6. 暫定結論

次に入れるべき「決算以外の運用制約」は、  
**まず同時保有数 / 1日新規建て数の上限** である。

理由:
- 効果が大きい
- 実装が比較的軽い
- 9銘柄版の実運用らしさを最も上げやすい

その次に、  
**手動停止フラグ** を入れるのが自然である。

現時点ではその次として、  
**`FY only block + EX_DIVIDEND_BLOCK_DAYS=1`** を
暫定候補として持つのがよい。
