# 日次紙上トレーダー 最小版

## 1. 目的

`daily_paper_monitor` の観察モードとは別に、  
毎日 1 回ずつ状態を引き継ぎながら

- 前日までの建玉を持ち越す
- 当日の stop / trailing を更新する
- 必要なら決済する
- 新規シグナルが出たら翌営業日寄りの買い予約を作る

ところまで進める。

## 2. 保存先

- 建玉: `live_positions`
- 約定: `live_trade_fills`
- 翌営業日予約: `daily_trader_orders`
- 日次レポート: `daily_monitor_reports`

## 3. 実行入口

```powershell
.\run_daily_paper_trader.ps1
```

指定日で進める例:

```powershell
.\run_daily_paper_trader.ps1 -TraderDate "2026-03-31"
```

## 4. 初期値

- `TraderSimulationName='paper_candidate9_daily'`
- `StrategyName='swing_v1_candidate9'`
- `MAX_OPEN_POSITIONS=5`
- `MAX_NEW_POSITIONS_PER_DAY=2`
- `EarningsBlockDays=2`
- `EarningsBlockLabels='FY'`
- `ExDividendBlockDays=1`

## 5. 1回の実行でやること

1. 当日寄りで、前営業日までに予約されていた `ENTRY / EXIT` を約定する
2. 当日足を使って保有中建玉を更新する
3. `TIME / REV` は翌営業日寄りの `EXIT` 予約を作る
4. 当日引け時点で signal が出た銘柄に、翌営業日寄りの `ENTRY` 予約を作る
5. 当日分の日次レポートを保存し、LINE 通知する

## 6. 注意

- 初版は、厳密な intraday 運用ではなく「引け後に1日分を進める」紙上運用
- 権利落ち回避は `ex_dividend` を近似として使う
- 初回は `LINE_CHANNEL_TOKEN` が入っていれば通知される
