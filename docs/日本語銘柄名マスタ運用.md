# 日本語銘柄名マスタ運用

## 方針

`tickers.name` は日本語優先で管理する。  
参照順は次の通り。

1. `jpx_ticker_master`
2. 検索 CSV
3. `yfinance`

## マスタテーブル

テーブル:
- `jpx_ticker_master`

主な列:
- `code`: `3103` のような生コード
- `ticker_code`: `3103.T` のようなアプリ側コード
- `name`: 日本語会社名
- `market_product_category`
- `source_date`
- `source_file`

## 取り込み

JPX の銘柄一覧 `xls` から、
- `プライム（内国株式）`
- `スタンダード（内国株式）`

だけを抽出して upsert する。

実行例:

```powershell
.\run_import_jpx_ticker_master.ps1 -XlsPath "C:\Users\suchida\Downloads\data_j .xls"
```

この実行で、
- `jpx_ticker_master` を更新
- `tickers.name` も日本語名で同期
- 参考用 CSV を `docs/jpx_ticker_master_prime_standard_latest.csv` に出力

## 表示用の軽い正規化

取り込み時に次の軽い正規化を行う。

- Unicode `NFKC` 正規化
- 連続空白の 1 個化
- `HLDG` / `HLDGS` を `HD` へ統一

これにより、全角英数字や全角空白が混ざっても、表示は少し読みやすくなる。

## 検索 CSV の補完

補完に使いたい CSV がある場合は、環境変数 `TICKER_NAME_SOURCE_CSVS` にカンマ区切りで指定する。

例:

```powershell
$env:TICKER_NAME_SOURCE_CSVS = "C:\Users\suchida\Documents\margintr\data20260324.csv"
```

## 補足

`jpx_ticker_master` が整っていれば、日本株の通常運用では検索 CSV 補完や `yfinance` 参照はほぼ補助用途になる。
