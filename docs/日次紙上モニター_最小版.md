# 日次紙上モニター 最小版

## 1. 目的

毎日、

- 新規候補
- 見送り理由
- 保有中建玉
- 直近イベント

を 1 枚で確認できるようにする。

これは実売買そのものではなく、  
`candidate9` を中心に「今日ならどう見えるか」を毎日観察するための最小レポートである。

## 2. 実行入口

```powershell
.\run_daily_paper_monitor.ps1
```

保存と LINE 通知まで行うジョブ版:

```powershell
.\run_daily_paper_monitor_job.ps1
```

指定日で見る例:

```powershell
.\run_daily_paper_monitor.ps1 -AsOf "2026-03-31"
```

保有中建玉も見たい場合:

```powershell
.\run_daily_paper_monitor.ps1 -AsOf "2026-03-31" -PositionSimulationName "paper_candidate9_clean_fy_exdiv_block1"
```

## 3. 初期値

- `StrategyName='swing_v1_candidate9'`
- `EarningsBlockDays=2`
- `EarningsBlockLabels='FY'`
- `ExDividendBlockDays=1`
- `UpcomingEventDays=7`

## 4. 出力先

- `docs/daily_reports/daily_paper_monitor_<strategy_name>_<yyyy-mm-dd>.md`
- DB 保存先: `daily_monitor_reports`

## 5. 見るもの

### Open Positions

- 指定 simulation の `OPEN` 建玉
- 含み損益
- stop / take
- `BREAK_EVEN` / `TRAILING` 状態
- 直近イベント

### New Entry Candidates

- 当日時点で signal が出ている銘柄
- 翌営業日の買付候補
- 直近イベント

### Blocked Signals

- signal は出ているが、運用制約で見送る銘柄
- 例:
  - `FY決算回避`
  - `権利落ち回避`
  - `手動停止`

### Upcoming Events

- `UpcomingEventDays` 以内の `FY決算` と `権利落ち`

## 6. 注意

- 初版は「毎日見るための観察レポート」であり、DB へ売買を進める処理はしない
- 権利落ち回避は `ex_dividend` を近似として使う
- 保有中建玉は `PositionSimulationName` を指定した時だけ表示する

## 7. ジョブ版

`run_daily_paper_monitor_job.ps1` は次をまとめて行う。

- レポート生成
- `daily_monitor_reports` への保存
- LINE 通知

LINE の送り方は `daily_task.py` と同じで、

- `LINE_USER_ID` があれば `push`
- なければ `broadcast`

を使う。
