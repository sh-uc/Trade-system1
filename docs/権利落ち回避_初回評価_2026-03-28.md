## 暫定結論

- `ex_dividend` を使った権利落ち回避は、最小実装として成立する
- `force close` は現時点ではやや強すぎる
- `block only` は試す価値がある
- 特に `7013.T（IHI）` では、`block` が悪化ではなく改善として効いた

## 感度確認: `EX_DIVIDEND_BLOCK_DAYS=1/2/3`

比較対象:

- `paper_candidate9_clean_fy`
- `paper_candidate9_clean_fy_exdiv_block1`

- `block 2`: `1,588,691.24`
- `block 3`: `1,588,691.24`

3本とも完全に同一だった。

止まった建玉も同じ:

- `7013.T（IHI）`
  - `2025-09-29` open
  - `2025-10-02` open
  - `2025-10-07` open

したがって、現時点のデータでは
`EX_DIVIDEND_BLOCK_DAYS` は `1 / 2 / 3` のどれでも差がない。

運用上は、最も軽い `1` を採ってよい。

## 暫定採用案

まずは次の軽い形が自然:

- `FY only block`
- `EX_DIVIDEND_BLOCK_DAYS=1`
- `EX_DIVIDEND_FORCE_CLOSE_DAYS=0`

今後、正式な `権利付最終日` ソースが入ったら、
`ex_dividend` 近似から差し替える。
