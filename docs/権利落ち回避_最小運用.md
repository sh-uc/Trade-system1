# 権利落ち回避 最小運用

## 1. 目的

`paper simulation` に、配当や権利落ちの前後を避ける最小ルールを入れる。

初版では、厳密な `権利付最終日` ではなく、`ticker_events.event_type='ex_dividend'` を
実務上の近似として使う。

## 2. 初版でできること

- `EX_DIVIDEND_BLOCK_DAYS`
  - `ex_dividend` が近い銘柄は新規建てしない
- `EX_DIVIDEND_FORCE_CLOSE_DAYS`
  - `ex_dividend` が近い保有建玉は、当日引けで `EX_DIVIDEND` 理由で強制決済する

## 3. 実行例

新規停止だけ確認する場合:

```powershell
.\run_simulate_live.ps1 -SimulationName "paper_candidate9_exdiv_block" -StrategyName "swing_v1_candidate9" -Start "2025-01-01" -Reset "1" -ExDividendBlockDays "2"
```

強制決済まで確認する場合:

```powershell
.\run_simulate_live.ps1 -SimulationName "paper_candidate9_exdiv_force" -StrategyName "swing_v1_candidate9" -Start "2025-01-01" -Reset "1" -ExDividendBlockDays "0" -ExDividendForceCloseDays "1"
```

## 4. 注意

- `ex_dividend` は `権利落ち日` に近く、`権利付最終日` そのものではない
- 初版では、安全側に倒す簡易回避ルールとして使う
- 将来、正式な `権利付最終日` や `record_date` が取れるソースを入れたら差し替える

## 5. 現時点での位置づけ

- 決算回避の次に入れる軽いイベント制約
- まずは `block only` で十分
- `force close` は、実際の影響と過剰回避のバランスを見ながら使う
