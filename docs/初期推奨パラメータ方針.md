# 初期推奨パラメータ方針（フェーズ1）

この文書は、`strategy_param_sets` に最初に入れる推奨パラメータセットの方針をまとめたものです。

---

## 1. 方針

初期投入では、次のルールを採用する。

- 対象は現時点の有力銘柄に限定する
- 1銘柄につき 1 つの active param set から始める
- 根拠になる `backtests_runs.id` を可能な限り残す
- `params` には実運用シミュレーションで必要な項目を明示的に入れる
- `STOP_PCT` は未指定にせず、現時点の設計意図が分かるよう明示する

---

## 2. 初期対象銘柄

フェーズ1の初期対象は以下とする。

- `3103.T`
- `7013.T`
- `4506.T`
- `6235.T`
- `6330.T`

理由:
- 現時点で有力と判断できる
- 候補群の中でも比較対象として使いやすい
- `paper simulation` の最初の対象数として多すぎない

## 4. 運用ルール

- `strategy_name` はまず `swing_v1` を使う
- `param_set_name` は `best_2026-03-17` のように作成日入りにする
- 新しい最適条件が見つかったら、旧 row を `is_active=false` にし、新 row を追加する
- update で上書きせず、履歴を残す

---

## 5. 次のステップ

1. `strategy_param_sets` に初期 row を insert する
2. `paper simulation` では active row のみ読む
3. 銘柄追加時は同じフォーマットで row を足す
